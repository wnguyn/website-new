I had an issue on Linux where that all the music players are worse than Windows. I use Navidrome because I don't like Spotify, but there aren't any good music players that have a decent UI for a large library. Granted, I only use Linux because WSL is too terrible and I'm not in the tax bracket to use MacOS: I don't really care about muh' heckin' FOSS software.

Anyways, my solution as a bandaid was MPD, since there were more options that weren't Electron or Tauri for a client. However, there weren't any support for the Subsonic API that worked for modern versions of MPD, so I thought of writing of a program just like MPD that would write on the same UNIX socket. Since MPD is just a simple socket, it really isn't hard to.. just reimplement all of the commands on mpd.readthedocs.io

Right now, it works and is about 2-3K LOC, but doesn't contain much of the nuanced features that you don't really need for playback. While I didn't vibe-code it, that doesn't really mean it's great code either. However, it's good enough for my own personal use.
